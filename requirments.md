## 已实现功能
- 支持配置多个宿舍作为查询对象，每个宿舍包含楼栋号、宿舍号、备注名称和收件人列表，查询和告警按宿舍独立处理。
- 支持手动立即查询，也支持后台定时查询并持续运行。
- 定时策略支持两种模式：按固定间隔分钟轮询，或者按每日指定整点查询；默认配置同时支持两者，实际运行时优先使用整点模式。
- 支持查询结果持久化到 SQLite 数据库，保存历史抄表记录、最近状态和缓存状态文件。
- 支持剩余电量预测：可按最近 N 次历史记录自动推算日均消耗，也可手动指定自定义日消耗量。
- 支持低电量提醒和预计耗尽提醒，当前告警渠道为邮件通知。
- 已完成 OpenWrt LuCI 控制台，支持通过 UCI 配置生成 JSON 配置，并打包为单个 ipk 安装包。

## 当前命令与接口
- `run`：启动后台监控服务。
- `query-now`：立即查询所有已配置宿舍，并写入历史记录。
- `status`：读取当前状态摘要，优先返回缓存结果。
- `test-notify`：发送测试邮件，用于验证邮件通知配置是否有效。

## 配置项说明
### 宿舍配置
- `building`：楼栋号。
- `room`：宿舍号。
- `label`：显示名称或备注。
- `recipients`：该宿舍对应的邮件收件人地址列表。

### 定时配置
- `interval_minutes`：固定轮询间隔，单位分钟。
- `check_hours`：每日执行查询的整点小时列表，例如 `8, 12, 18, 22`。

### 预测配置
- `sample_count`：参与预测计算的历史记录数量。
- `custom_daily_consumption`：自定义日消耗量，单位度/天；设为 0 时自动根据历史记录估算。

### 告警配置
- `low_energy_threshold`：低电量预警阈值，剩余电量低于该值时触发提醒。
- `depletion_alert_days`：预计耗尽时间距离当前不足多少天时触发提醒。
- `max_records_per_room`：每个宿舍最多保留的历史记录数；设为 0 表示不限制。
- `email`：SMTP 服务器、用户名、密码、发件人和收件人配置。

## 核心实现思路与工具包
### HTTP 查询
使用 `net/http` 发送 POST 表单请求，使用 `encoding/json` 解析返回的 JSON 数据，使用 `net/url` 处理表单参数。

### 定时任务
使用 `time.NewTicker()` 实现后台轮询。程序启动后先执行一次全量查询，再根据配置进入固定间隔模式或整点模式。

### 电量预测
根据最近若干次抄表记录计算日均消耗速度，再结合最新剩余电量推算预计耗尽时间；如果配置了自定义日消耗量，则直接使用该值计算。

### 通知功能
邮件通知，使用 `github.com/jordan-wright/email` 结合标准库 `net/smtp` 发送告警邮件和测试邮件。
