# LuCI App 5echarging-go | 抑疑电止

OpenWrt LuCI 电量监控插件，用于查询和记录宿舍剩余电量，并在低电量/预计耗尽时发送邮件提醒

## 功能

- 多独立配置: 楼栋 | 宿舍号 | 备注 | 收件邮箱
- 手动查询
- 定时查询(整点查询/定时查询)
- 可自定义数据库路径，默认 `/etc/5echarging/5echarging.bbolt`
- 耗尽时间预测，计算依据: 自定义固定日消耗 | 根据历史记录计算日消耗
- 检测充值并按充值周期分段计算，多个有效分段按时间跨度加权平均
- 插入数据库前去重
- 每宿舍历史记录数上限，默认500
- 低电量(自定义阈值)&预计耗尽几天前(自定义)发送邮件告警

## 快速开始

从本项目的 Releases 下载最新 ipk，上传至 OpenWrt 安装。
随后在`服务`一栏找到`抑疑电止`，选择对应架构在线下载核心，或提前下载核心后离线上传安装。核心就绪后，勾选启用服务，填写好参数信息后保存并应用即可。

如果设备在认证前无法访问 GitHub，请从 Releases 下载对应的 `5echarging-linux-*` 核心文件，进入 LuCI 后使用“上传并安装核心”。

当前在线发布的核心架构为 `amd64`、`arm`、`arm64`、`mips`、`mipsle`。

## 数据库

默认数据库位置：

```text
/etc/5echarging/5echarging.bbolt
```

写入策略：

- 查询成功后和同一宿舍最新记录比较
- 若余额相同，且抄表时间也相同，则跳过写库
- 如果余额或抄表时间变化，则写入新历史记录
- 写入后若超出`最大历史记录数`则裁剪最旧记录

历史记录使用 bbolt 存储。旧版 SQLite 数据库不会自动迁移，新版本会从新的 `.bbolt` 数据库重新开始采样。

## 预测逻辑

自动预测使用最近数条历史记录(由`历史采样次数`决定)：

- 先按时间排序
- 检测余额上升作为充值点，并按充值切分为多个消耗段
- 每个分段内部计算消耗量和时间跨度
- 多个有效分段按时间跨度加权平均，得到日均消耗
- 如果配置了`自定义日消耗`，则不做预测直接使用该值

若最近样本没有观察到电量下降，预计耗尽时间会显示为不可用。这可能发生在刚充值、长期无人用电、或接口抄表数据尚未更新时

## 构建

本仓库的 ipk 是通用 LuCI 套壳包（`Architecture: all`），不包含 Go 核心二进制。构建 ipk 只需要一个可用的 OpenWrt SDK；核心二进制由 GitHub Actions 发布到 Releases，或本地使用 Go 交叉编译。

获取与你的 OpenWrt 主版本匹配的[OpenWrt SDK](https://downloads.openwrt.org)

1. 将 SDK 解压至项目根目录
2. 在项目根目录执行：

```bash
chmod +x ./build.sh
./build.sh
```

脚本会自动打包 LuCI 套壳，完成后只需在项目根目录寻找 `luci-app-5echarging_*_all.ipk`
若报错请手动补齐依赖

如需指定 SDK 路径：

```bash
./build.sh /path/to/openwrt-sdk-24.xx
```

## 注意事项

- 卸载插件不会主动删除数据库，如有需要请手动删除
- 长时间停用后再启用，历史跨度可能影响预测，后续持续采样后预测会逐步恢复到当前用电状态
